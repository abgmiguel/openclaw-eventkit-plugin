# Contributing

## Goal

Keep these docs implementation-accurate and optimized for fast onboarding.

## Rules

- Prefer concrete behavior over design intent.
- Document exact field names and command names.
- Keep examples copy-paste ready.
- Keep changes incremental and aligned with existing OpenClaw architecture.
- Do not document commands that do not exist in product code.

## Update Process

1. Update docs for the code change.
2. Add or update examples in `examples/`.
3. Validate referenced command names and fields against source.
4. Include a short changelog entry.

## Style

- ASCII only.
- Use clear, direct language.
- Keep sections short and scannable.
